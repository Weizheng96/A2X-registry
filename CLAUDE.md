## Project Overview

A2X Registry validates that **hierarchical taxonomy + LLM recursive retrieval** achieves more accurate service discovery than semantic vector retrieval.

**Core hypothesis**: Explicit category structure guides LLM through multi-step reasoning, decomposing complex "query→service" mapping into simpler "query→category" decisions, avoiding the semantic gap problem of vector retrieval.

### Backend Key Patterns

**Single SearchService singleton** — created at module level in `search_service.py`:
```python
search_service = SearchService()  # all routers import this shared instance
```

**Per-dataset embedding models**: `SearchService` caches `EmbeddingModel` instances by model name (not by dataset). Each dataset reads its model from `vector_config.json`. Model mismatch during `sync_vector()` triggers a full ChromaDB collection rebuild.

**4 routers**: `dataset.py` (CRUD, services, taxonomy, embedding), `build.py` (taxonomy build + SSE), `search.py` (search + judge), `provider.py` (LLM switching). No router-to-router dependencies — all share `search_service` singleton and `RegistryService` instance.

**Startup sequence** — `startup.py:run_warmup()` runs in a background thread. Warmup state polled via `GET /api/warmup-status`.

**TaxonomyState machine**: `nonexistent → available` (after build) → `unavailable` (after service change) → re-check on next search call. State is determined by comparing service.json content hash against `build_config.json`.

**Vector incremental sync**: After register/deregister, a daemon thread syncs ChromaDB using the dataset's configured embedding model — only re-encodes changed descriptions, removes deleted IDs, and invalidates the cached VectorSearch instance.

## Commands

### Build
```bash
# Full rebuild (output-dir defaults to database/{dataset_name})
python -m src.a2x.build --service-path database/ToolRet_clean/service.json

# Resume from checkpoint (skip if complete, continue if partial, rebuild if config mismatch)
python -m src.a2x.build --service-path database/ToolRet_clean/service.json --resume yes

# Rebuild but reuse cached keywords (for debugging/iteration)
python -m src.a2x.build --service-path database/ToolRet_clean/service.json --resume keyword

# Optional: --generic-ratio 0.333, --delete-threshold 2, --no-cross-domain, --max-depth 1
```

### Register
```bash
# Load config and generate service.json for all datasets
python -m src.register --config src/register/user_config.example.json

# Show registry status
python -m src.register --status
python -m src.register --status --dataset publicMCP
```

### Evaluate
```bash
# A2X (output dir auto-generated: results/{date}_a2x-{mode}_{dataset}_{count})
python -m src.a2x.evaluation --data-dir database/ToolRet_clean --max-queries 50 --workers 20
python -m src.a2x.evaluation --data-dir database/ToolRet_clean --workers 20 --mode get_one
python -m src.a2x.evaluation --data-dir database/ToolRet_clean --workers 20 --mode get_important

# Vector
python -m src.vector.evaluation --max-queries 50

# Traditional (MCP-style)
python -m src.traditional.evaluation --max-queries 50
```

## Taxonomy Format

- `taxonomy.json`: `{version, root, build_status, categories: {id: {children[], services[]}}}`
  - `build_status`: `"bfs"` (splitting in progress) | `"cross_domain"` (BFS done) | `"complete"`
- `class.json`: `{categories: {id: {name, description}}}`
- Supports multi-parent (same service in multiple categories)
- Dataset naming convention: parent directory of `service.json` is the dataset name (e.g., `database/ToolRet_clean/service.json` → dataset `ToolRet_clean`)

## Key Classes

**A2XSearch** (`src/a2x/search/a2x_search.py`):
- `A2XSearch(taxonomy_path, class_path, service_path, max_workers, parallel, mode)` — mode: `"get_all"` | `"get_important"` | `"get_one"`
- `search(query, stream=False)` → (List[SearchResult], SearchStats) or Generator (stream=True)
- Composed of: CategoryNavigator (navigator.py) + ServiceSelector (selector.py) + prompts (prompts.py)
- get_one auto-fallback to get_important when no results found

**VectorSearch** (`src/vector/search/vector_search.py`):
- `search(query, top_k)` → (List[SearchResult], SearchStats)

**TraditionalSearch** (`src/traditional/search/traditional_search.py`):
- `search(query)` → (List[SearchResult], SearchStats)

**LLMClient** (`src/a2x/utils/llm_client.py`):
- OpenAI-compatible API wrapper with retry and key rotation

**SearchService** (`src/backend/services/search_service.py`):
- Module-level singleton: `search_service = SearchService()` — import this, do not instantiate
- `search(query, method, dataset, top_k)` → dict (no streaming)
- `search_stream(query, method, dataset)` → Generator[dict] (A2X only)
- `judge_services(query, services)` → list[{service_id, relevant}]
- `switch_provider(name)` — resets all A2X/LLM instances, updates `_current_provider`
- `sync_vector(dataset)` / `schedule_vector_sync(dataset)` — ChromaDB incremental sync

**RegistryService** (`src/register/service.py`):
- `RegistryService(database_dir, global_config_path)` — multi-dataset service registration
- `startup()` → Dict[str, TaxonomyState] — load configs, fetch agent cards, generate service.json
- `register_generic(req)` / `register_a2a(req)` → RegisterResponse
- `deregister(dataset, service_id)` → DeregisterResponse
- `check_taxonomy_state(dataset)` → TaxonomyState | None (None = unmanaged dataset)

## Instructions

1. LLM API config: create `llm_apikey.json` locally (see `llm_apikey.example.json`)
2. Parallel LLM Calls: Always use `ThreadPoolExecutor` (default 20 workers)
3. **Test with 50 queries first**, then run full evaluation after confirming reasonable results
4. **Never manually edit taxonomy.json / class.json** — only modify build code
5. **不在文档中引用 get_one / get_important 模式的评估指标**：这两个模式找到的服务几乎都与请求功能相关（人工抽样功能精度 ~98%），但当前 ground truth 不完整（未标注所有等价/相关服务），导致 strict Precision/Recall 严重低估实际效果。在 ground truth 完善之前，这些评估数字容易引起误解。

## Design Principles

- Hierarchical taxonomy with functional orientation
- Multi-parent support (services can belong to multiple categories)
- Generalization first (avoid overfitting to current dataset)
